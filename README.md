# games-homepage-layout

## 基础信息

- 提交人：xiashiyu.09
- 维护人：xiashiyu.09
- 来源：互动
- 所属业务域：营销域
- 当前状态：进行中

## 用途

这个 Skill 用于把已确认的营销互动 PRD 事实清单转换为首页低保真布局决策，明确页面骨架、标准槽位、出现状态、组合关系、首屏排布、安全避让和 3px 间距网格。

## 适用场景

- 已完成 PRD 事实提取，需要进入首页低保真布局判断。
- 需要判断首页适合全屏沉浸型、半屏效率型还是三分屏内容型。
- 需要确定顶部通栏、核心玩法区、常驻功能信息区、资源位、Feeds 等槽位是否出现及如何组合。

## 输入

- `games-prdbrief` 输出并经用户确认的 PRD 事实清单。
- 页面目标、核心转化动作、玩法目标、表现载体、用户动作和关键状态信息。
- 需要展示的任务、奖励、资源位、Feeds 或常驻入口要求。

## 输出

- 推荐页面骨架。
- 槽位出现状态和优先级。
- 槽位组合关系。
- 正常纵向顺序和独立槽位间距。
- 页面边距、安全避让区和叠加说明。
- 供 `games-lofi` 使用的内部首页布局结果。

## 依赖资料

- `SKILL.md`：首页布局判断规则与固定输出表。
- `references/rules.md`：骨架判断、槽位关系、间距和安全避让规则。
- `games-prdbrief`：PRD 事实输入。
- `games-component-specs`：按需查询组件尺寸、状态和可复用规则。

## 边界和风险

- 不得跳过 PRD 事实确认直接生成布局。
- 不得因为槽位清单中存在某模块就默认画进首页。
- 不得用非标准槽位名称替代本 Skill 中定义的标准名称。
- 叠加、贴附、悬浮关系必须声明安全区和点击热区。
- 布局决策会影响后续低保真、高保真和动效阶段，关键业务模块需由业务方确认。
