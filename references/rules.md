# 首页布局决策规则

## 目标

把已确认的 PRD 事实清单转换为可执行的首页骨架、槽位状态和组合关系。

## 判断顺序

必须按以下顺序执行，不得直接堆模块：

`选择页面骨架 -> 判断槽位是否出现 -> 定义槽位关系 -> 排布首屏 -> 检查遮挡`

## 标准名称

1. 使用 `头图氛围区`，不写“背景氛围”。
2. 使用 `核心玩法区`，不写“核心游戏区”。
3. 使用 `辅按钮`，不写“副按钮”。
4. 使用 `资源位`，不写“资源位卡片”。
5. 使用 `Feeds`，不写“商品 Feeds”。
6. 账户、等级、体力、次数、资产等信息统一归入 `常驻功能信息区`。

## 骨架规则

- A. 全屏沉浸型：适合单一核心玩法、强场景感、首页即游戏主界面。
- B. 半屏效率型：适合上半屏完成玩法或转化，下半屏承接任务、奖励、资源位或 Feeds。
- C. 三分屏内容型：适合活动信息、核心操作、内容承接同等重要。

核心玩法区高度按首屏可用高度计算：

`首屏可用高度 = 首屏高度 - 顶部通栏实际占用高度 - 底部固定区域实际占用高度`

## 槽位规则

每个槽位必须标记一种状态：

- `必选`：缺失会导致核心流程无法完成。
- `条件出现`：满足业务、状态或用户条件时出现。
- `可选`：用于增强理解、氛围或转化。
- `不出现`：本页明确不使用。

不得因为槽位清单中存在某模块，就默认画进首页。

## 组合关系

强关联槽位必须选择一种关系：

- `独立`
- `包含`
- `叠加`
- `贴附`
- `悬浮`
- `融合`

使用叠加、贴附或悬浮时，必须写明基准槽位、对齐边、覆盖方向、最大覆盖范围、点击热区和安全避让区。

## 间距规则

1. 独立槽位外部间距统一使用 3px 网格。
2. 页面左右安全边距默认 `30px`。
3. 相邻独立槽位纵向和横向间距必须为 3 的倍数。
4. 独立槽位不得使用 `0px` 间距；无间距时改用包含、叠加、贴附或融合关系。
5. 单处间距不得超过 `99px`。

## 决策优先级

出现冲突时按以下顺序决策：

`核心玩法可理解 -> 核心操作可完成 -> 状态信息可见 -> 营销利益点清晰 -> 内容丰富度`
